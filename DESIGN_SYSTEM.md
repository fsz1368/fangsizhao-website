# 方思照个人网站 — 视觉升级方案 v3

> 基调：冷色深空 + 暖金点缀 · 现代极简 · 移动端优先 · 非模板感

---

## 一、配色方案

**选择**：冷色系主基调。深空蓝黑承载无人机/测绘的技术感，暖金作为唯一强调色呼应新媒体/运营的创造力。单一强调色比多色方案更具品牌识别度。

### 完整色板

| 角色 | 色名 | HEX | 使用场景 | 占比 |
|------|------|-----|----------|------|
| 主背景 | Deep Navy | `#0A0E17` | 全局暗色背景 | 55% |
| 卡片/区块背景 | Slate | `#111620` | 卡片、时间线、表单 | 25% |
| 主文字 | Frost White | `#ECEFF4` | 正文、标题 | 12% |
| 辅助文字 | Arctic Gray | `#7C8798` | 日期、描述、标签 | 5% |
| 强调色 | Amber Gold | `#E8990C` | 按钮、链接、高亮、焦点 | 2% |
| 技术标签 | Glacier Blue | `#5E9EFF` | 技能标签、代码、数据 | 1% |

### 亮色模式对应

| 角色 | HEX |
|------|-----|
| 背景 | `#F5F6F8` |
| 卡片 | `#FFFFFF` |
| 主文字 | `#0F1419` |
| 辅文字 | `#5B6778` |
| 强调色 | `#C47D08`（同 Amber 加深） |
| 技术标签 | `#2563EB` |

### 使用规则

- Amber Gold 每屏最多出现 2 处，避免稀释强调效果
- Glacier Blue 仅用于技能/数据/代码等技术语义场景
- 渐变只在一处使用：Hero 标题文字（Frost White → Amber Gold 135deg）
- 禁止使用纯黑 `#000000` 和纯白 `#FFFFFF`

## 二、字体层级系统

### 字体族

```css
font-family: "Inter", "PingFang SC", "Microsoft YaHei", -apple-system, sans-serif;
font-family-mono: "JetBrains Mono", "SF Mono", "Cascadia Code", Consolas, monospace;
```

西文优先 Inter（现代几何风格，x-height 高，屏幕可读性优秀），中文回退苹方/微软雅黑。等宽字体给数据/代码场景。

### 层级表

| 层级 | 标签 | 字号 | 字重 | 行距 | 字间距 | 用途 |
|------|------|------|------|------|--------|------|
| H1 | `.hero-name` | 3.5rem (56px) | 800 | 1.1 | -0.02em | 首页姓名 |
| H2 | `.section-title` | 2rem (32px) | 700 | 1.2 | -0.01em | 板块标题 |
| H3 | `.card-title` | 1.125rem (18px) | 600 | 1.3 | 0 | 卡片/时间线标题 |
| Body L | `.lead` | 1.125rem (18px) | 400 | 1.75 | 0 | Hero 副标题 |
| Body | `p` | 0.9375rem (15px) | 400 | 1.7 | 0 | 正文 |
| Small | `.text-secondary` | 0.8125rem (13px) | 400 | 1.5 | 0.01em | 日期/标签 |
| Code | `code` | 0.875rem (14px) | 500 | 1.5 | 0 | 数据数字 |
| Caption | `.section-tag` | 0.6875rem (11px) | 700 | 1 | 0.15em | 板块英文标签 |

### 响应式缩放

```css
@media (max-width: 768px) {
  .hero-name { font-size: 2.25rem; }
  .section-title { font-size: 1.5rem; }
  .lead { font-size: 1rem; }
}
```

## 三、六大板块布局重构

### 板块 1 · Hero（首页首屏）

**视觉重心**：姓名（H1，渐变文字）+ 一句话定位（Body L），居中垂直排列。

**线框结构**：
```
┌──────────────────────────────┐
│          (Canvas 背景)        │
│                              │
│         [低多边形无人机]      │  ← 浮动动画，不遮挡文字
│                              │
│         方  思  照            │  ← H1 56px 渐变字
│    无人机 × 新媒体 双栖实践者  │  ← 打字机效果，18px Amber
│                              │
│   [查看作品]   [联系我]       │  ← 两个按钮，间距 16px
│           ↓                  │  ← 滚动指示箭头
└──────────────────────────────┘
```
- 排版：绝对居中对称
- 图文比：100:0（纯文字+Canvas 装饰）
- 高度：`100vh`，内容垂直居中

## 四、交互微动效

### 4.1 悬停反馈

| 元素 | 属性变化 | 时长 | 缓动 |
|------|---------|------|------|
| 按钮（btn） | `transform: scale(0.96)` + `box-shadow` 加深 | 150ms | `ease-out` |
| 卡片（card） | `transform: translateY(-4px)` + `box-shadow` 扩大 | 300ms | `cubic-bezier(0.25, 0.8, 0.25, 1.2)` |
| 导航链接 | `color: Amber` + 下划线 `scaleX(0→1)` | 200ms | `ease` |
| 技能标签 | `transform: translateY(-1px)` + 边框变亮 | 150ms | `ease-out` |
| 联系卡片 | `transform: translateY(-6px)` + 图标背景变 Amber | 350ms | `cubic-bezier(0.25, 0.8, 0.25, 1.2)` |

### 4.2 滚动渐现动画

```
触发方式：IntersectionObserver，threshold: 0.1，rootMargin: 0px 0px -30px 0px
基础动画：opacity (0→1) + translateY (30px→0)
时长：700ms
缓动：cubic-bezier(0.25, 0.8, 0.25, 1)
序列延迟：每子元素递增 100ms
```

| 板块 | 子元素序列 |
|------|-----------|
| Hero | 无（始终可见） |
| About | 视频先 → 文字 100ms → 数据卡片 200ms |
| Skills | 雷达图 → 卡片1(100ms) → 卡片2(200ms) → 卡片3(300ms) |
| Experience | 条目1 → 条目2(100ms) → 条目3(200ms) |
| Portfolio | 工具栏先 → 网格项逐行 50ms/行 |
| Contact | 三卡同时 → 下载按钮 150ms |

### 4.3 页面过渡

| 场景 | 类型 | 时长 | 缓动 |
|------|------|------|------|
| 灯箱打开 | `opacity: 0→1` + 背景 `blur(0→8px)` | 350ms | `ease-out` |
| 灯箱关闭 | 同上反向 | 250ms | `ease-in` |
| 主题切换 | `background-color` + `color` 全页面过渡 | 300ms | `ease` |
| 导航显隐 | `transform: translateY` 滑入/滑出 | 400ms | `cubic-bezier(0.25, 0.8, 0.25, 1)` |
| 上传进度条 | 伪元素 `animation: uploadPulse 1.5s ease-in-out infinite` | — | — |

## 五、图文排版比例

| 板块 | 图:文 | 设计意图 |
|------|-------|----------|
| **Hero** | 0:100 | 纯文字+Canvas装饰，最大化个人品牌聚焦，无人机动画做氛围而非信息载体 |
| **About** | 40:60 | 视频先抓注意力(40%)，文字叙事建立情感连接(60%)。视频短循环降低阅读负担 |
| **Skills** | 30:70 | 雷达图快速传达"多维度能力"(30%)，标签卡片提供具体细节(70%) |
| **Experience** | 5:95 | 时间线竖线仅做视觉引导，内容本身才是信息核心 |
| **Portfolio** | 95:5 | 作品本身即内容，文字仅用作分类标注和简短说明 |
| **Contact** | 10:90 | 图标提示交互意图，文字提供实际信息 |

### 全局比例规则

- 任何一屏中，留白面积 ≥ 30%（通过 `padding: 100px 0` + 容器 `max-width: 1140px` 保证）
- 段落宽度 ≤ 65ch（约 600px），防止行长过长降低可读性
- 卡片最小高度 ≥ 120px，避免内容稀疏时视觉塌陷
